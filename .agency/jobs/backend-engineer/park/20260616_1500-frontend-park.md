### Infinite review loop

**Problem**: review deck was `reviewWords` (SRS-due words). When empty, showed "No words due for review." Felt like a task with an endpoint.

**Change**: replaced `reviewWords` as the card source with a `reviewDeck` state — a shuffled copy of all `vocabulary`.

Key details:
- `deckInitialized` ref (useRef) ensures the deck is built once from `vocabulary` after fetchData resolves. Guarded by `!deckInitialized.current` so subsequent vocabulary changes (word deleted) don't re-shuffle mid-session.
- End of deck in `handleReviewAnswer`: `setReviewDeck([...vocabulary].sort(() => Math.random() - 0.5))` then reset to card 0 — no toast, no pause.
- Empty state condition: `vocabulary.length === 0` (only when truly nothing saved) → "Save some words while reading to start reviewing them."
- `getReviewWords()` still fetched (used in tab badge count). `submitReview` still called on every answer (SRS tracking unchanged).
- All `reviewWords` references in review tab replaced with `reviewDeck` (progress counter, dot indicators, navigation buttons, card display).
