1. **Atomic decrement race condition** (flagged as required): When Block 2/3 implements the `spots_remaining` decrement, it must be atomic. The naive approach (`find` then `update`) has a race condition if two users hit checkout simultaneously. Recommend MongoDB's `findOneAndUpdate` with `$inc: { spots_remaining: -1 }` plus a filter `{ spots_remaining: { $gt: 0 } }` — if the update matches 0 documents, the spot was taken. COO should confirm this approach is acceptable before Block 2/3 implementation starts.
