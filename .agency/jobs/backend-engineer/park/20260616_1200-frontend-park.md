### Commit 1 — `categoryColors.js` (cd217f8)

**Colors strengthened** across all categories:
- `strip`: `-500` → `-600` (deeper, richer bar)
- `bg`: `bg-{color}-50` → `bg-{color}-100` (visibly tinted, not nearly-white)
- `badge`: `-100/-700` → `-200/-900` (dark ink text on clearly colored chip)

**`particle` category added** (teal-600) for は/が/を/に etc.

**`posToCategory()` exported** — maps UniDic `feature[0]` POS values:
```
動詞 → verb
名詞 → noun
形容詞 / 形状詞 → adjective
助詞 → particle
副詞 / 感動詞 / 接続詞 → expression
everything else → other
```

### Commit 2 — Click chain + picker removal (74e414e)

**`ReaderPage.jsx` — `TokenizedSentence` component (lines 101–119):**
Both click handlers changed from `onWordClick(token.surface, e)` to `onWordClick(token.surface, token.pos, e)`.

**`ReaderPage.jsx` — `handleWordClick` (line ~341):**
- Signature: `(word, event, sentenceText = '')` → `(word, pos, event, sentenceText = '')`
- Added `setWordPos(pos || '')` in body
- New `wordPos` state added to dictionary popup state block

**Call site wrappers in `ReaderPage.jsx`:**
- `HighlightedText` wrapper: `handleWordClick(word, '', e, getSentenceText(sentence))` — passes `''` for pos (no token data in highlight mode)
- `TokenizedSentence` wrapper: `onWordClick={(word, pos, e) => handleWordClick(word, pos, e)}`
- `DictionaryPopup` render: added `pos={wordPos}` prop

**`DictionaryPopup.jsx`:**
- Added `pos = ''` prop, imported `posToCategory`
- Removed `selectedCategory` state entirely
- Removed pill-button picker UI block
- Save call: `category: posToCategory(pos)` (auto-assigned, silent)
- `displayCategory` fallback: `posToCategory(pos)` (for the just-saved chip)

**Why `HighlightedText` passes `''`**: The highlighted-text path operates on pre-segmented vocab matches, not MeCab tokens — no POS data is available there. Words saved from that path fall back to `'other'`. The key path is `TokenizedSentence` (highlights OFF), which always has `token.pos`.

### Commit 3 — Vocabulary filter tabs (ac8a831)

**State rename**: `typeFilter` / `setTypeFilter` → `activeFilter` / `setActiveFilter`

**`FILTER_TABS` array** defined (scoped inside component):
`All | Words | 漢字 | Verbs | Nouns | Adjectives | Particles | Expressions | Other`

**Filter logic** updated:
- `word` filter: `type === 'word'` (all words, any category)
- `kanji` filter: `type === 'kanji'`
- Category filters (verb/noun/etc.): `type === 'word' && getWordCategory(word) === activeFilter`
- `all`: no filter

**Active tab styling**: `Words` uses its own amber (type-level, not category-level). All category tabs use `CATEGORY_COLORS[value].badge`. `All` uses neutral secondary.

**Empty state**: when a category filter is active and produces zero results with no search query, shows "No {label.toLowerCase()} saved yet" (e.g. "No particles saved yet").

**`api-contracts.md`**: updated `category` field to include `"particle"` in the value set.
