## What was done

### Category field + stats placeholders

Added `category` field and stats placeholder fields to `SaveWordRequest`, `SavedWordResponse`, and the `save_word` endpoint's `word_doc`. Single commit, purely additive.

**`SaveWordRequest`** — added:
```python
category: Optional[str] = None  # "verb" | "noun" | "adjective" | "expression" | "other" | None
```

**`SavedWordResponse`** — added:
```python
category: Optional[str] = None
correct_count: int = 0
incorrect_count: int = 0
last_reviewed: Optional[str] = None
```

**`save_word` word_doc** — added to the dict passed to `insert_one`:
```python
"category": word_data.category,
"correct_count": 0,
"incorrect_count": 0,
"last_reviewed": None,
```

`times_reviewed` was already present in both the model and word_doc — not re-added.

**Documentation** — updated both on-demand docs:
- `db-schema.md`: `saved_words` table now includes `category` and the three stats placeholder fields, with usage notes
- `api-contracts.md`: Vocabulary section now includes full `SaveWordRequest` and `SavedWordResponse` shapes with all new fields documented

## Design decisions

**`last_reviewed` stored as `Optional[str]` (ISO8601), not `datetime`**: consistent with the rest of the codebase — all timestamps in `saved_words` are ISO8601 strings (`next_review`, `created_at`, etc.).

**Stats fields are placeholders only**: `correct_count` and `incorrect_count` are initialized to `0` in `word_doc` but the review endpoint (`POST /api/vocabulary/review`) only increments `times_reviewed` — it does not yet write `correct_count`/`incorrect_count`/`last_reviewed`. Those are for a future stats pass.

**Existing records**: documents without `category` have no `category` field in MongoDB. `SavedWordResponse` defaults `category` to `None` (Pydantic v2 with `ConfigDict(extra="ignore")`), which frontend treats as `"other"`. No migration needed.
